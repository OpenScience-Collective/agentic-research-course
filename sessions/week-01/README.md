# Week 1: Git, GitHub, and the Command Line

## Learning Objectives

By the end of this session, you will:
- Understand why version control matters for research reproducibility
- Navigate the terminal with basic commands
- Create a GitHub account and repository
- Use git to clone, commit, push, and pull
- Understand branching and pull requests

## Outline

### Part 1: Why Version Control? (10 min)
- The problem: "manuscript_final_v3_REAL_final.docx"
- How git solves this for code, data pipelines, and manuscripts
- Real examples from neuroscience research

### Part 2: Terminal Basics (10 min)
- Navigating directories: `cd`, `ls`, `pwd`
- Creating and editing files: `mkdir`, `touch`
- Installing tools: Homebrew (macOS), apt (Linux)

### Part 3: Git Fundamentals (15 min)
- `git init`, `git clone`
- The staging area: `git add`, `git status`
- Committing: `git commit -m "message"`
- Pushing: `git push`
- Pulling: `git pull`

### Part 4: GitHub Workflow (10 min)
- Creating a repository on GitHub
- Branches: `git checkout -b feature-name`
- Pull requests: why they matter
- Issues: tracking work

### Q&A (15 min)

## Key Concepts

- **Repository:** A project folder tracked by git
- **Commit:** A snapshot of your project at a point in time
- **Branch:** A parallel version of your project for experimentation
- **Pull request:** A proposal to merge changes from one branch to another
- **Remote:** The copy of your repo on GitHub (vs local on your machine)
