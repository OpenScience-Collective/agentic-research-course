# Documentation & Content Standards

## Session Content Structure
Each session folder (`sessions/week-NN/`) should contain:
- `README.md` -- learning objectives, outline, key concepts, pre-session prep
- `references.md` -- links to documentation, papers, tools (when applicable)
- `exercises.md` -- practice exercises for post-session work (when applicable)

## Presentation Structure
Each presentation folder (`presentations/week-NN/`) should contain:
- Slide outlines or structured content for live sessions
- Speaker notes where helpful
- Demo scripts or command sequences for live coding

## Writing Philosophy
- **Start with why:** Context before details
- **Show, don't tell:** Examples over explanations
- **Progressive disclosure:** Simple first, then advanced
- **Define abbreviations** on first use (e.g., "Brain Imaging Data Structure (BIDS)")
- **No em-dashes;** use commas or semicolons

## Content Principles
- Real tool output and real commands; no hypothetical examples
- Each session builds on prior sessions
- Practicum (HBN analysis) threads through sessions as a running example
- Key message: AI tools amplify productivity, so structured workflows and QA matter more

## README Template
```markdown

## Learning Objectives

- [Specific, measurable objective]

## Outline

## Key Concepts
- **Term:** Definition

## Pre-Session Prep
- [ ] Task to complete before attending

## Resources
- [Link](url) -- description
```
