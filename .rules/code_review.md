# Code Review Standards

## PR Review Process
**No technical debt carried forward.** Address all findings, not just critical ones.

1. Create PR with `gh pr create`
2. Run `/review-pr` on the changes
3. Address ALL findings: critical, important, suggestions
4. Only skip genuine false positives or intentional design choices
5. Document skipped findings with reasoning

## Content Review Checklist

### Before Committing Session Content
- [ ] Learning objectives are specific and measurable
- [ ] Outline matches the session time budget
- [ ] Key concepts are defined clearly
- [ ] Pre-session prep is actionable
- [ ] No broken links
- [ ] No undefined abbreviations

### Code Examples
- [ ] Commands actually work (tested)
- [ ] Output shown is real, not hypothetical
- [ ] Paths and tool names are current

---
*Review early, review often. No technical debt carried forward.*
