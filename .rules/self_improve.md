# Continuous Improvement

## Philosophy
Rules grow from experience. After each session, capture what worked and what didn't.

## Improvement Triggers
- Student question reveals a gap in materials -> update session content
- Live demo fails -> log in .context/scratch_history.md, add prevention notes
- Pattern used 3+ times -> create a template or rule
- Successful pedagogy approach -> document in .context/ideas.md

## After Each Session
1. Review what worked and what didn't
2. Update session README if objectives shifted
3. Log any technical issues in .context/scratch_history.md
4. Update .context/plan.md with next session prep tasks
